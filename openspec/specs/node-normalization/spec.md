# node-normalization Specification

## Purpose

Спецификация поведения нормализации узла и наблюдаемости процесса выполнения.

## Requirements

### Requirement: Сбор логов операции нормализации

Система SHALL собирать лог выполнения операции нормализации узла из `stdout` и `stderr` процесса Cursor Agent, а также добавлять системные события стадии операции (`normalize`, `sync`, `done/error`). Каждая запись MUST иметь монотонный offset, временную метку и тип потока.

#### Сценарий: Логи stdout/stderr во время normalizing

- **WHEN** операция нормализации запущена
- **THEN** записи вывода агента добавляются в лог операции с типом `stdout` или `stderr`

#### Сценарий: Системные записи стадий

- **WHEN** операция переходит между стадиями
- **THEN** в лог добавляется запись типа `system` с описанием перехода

### Requirement: Нормализация узла через Cursor Agent

Система SHALL предоставлять серверный сценарий нормализации конкретного узла базы знаний через Cursor Agent с промтом, сформированным на основе текущего содержимого узла и его метаданных. Операция MUST быть привязана к одному path узла и MUST возвращать диагностируемый статус выполнения.

#### Сценарий: Успешный запуск нормализации

- **WHEN** клиент запрашивает нормализацию существующего узла и runtime готов (Cursor Agent доступен)
- **THEN** сервер запускает задачу нормализации для этого path и возвращает идентификатор операции со статусом running

#### Сценарий: Узел не найден

- **WHEN** клиент запрашивает нормализацию для несуществующего path
- **THEN** сервер возвращает ошибку not found и MUST NOT запускать Cursor Agent

### Requirement: Автоматический sync после успешной нормализации

После завершения шага нормализации со статусом success система SHALL автоматически выполнять `sync` и MUST отражать итог обоих шагов в результате операции.

#### Сценарий: Нормализация и sync успешны

- **WHEN** Cursor Agent завершил нормализацию успешно и `sync` выполнился без ошибок
- **THEN** итоговый статус операции success, а изменения узла доступны в обычном git-first потоке

#### Сценарий: Нормализация успешна, sync завершился ошибкой

- **WHEN** нормализация завершилась success, но `sync` вернул ошибку
- **THEN** операция возвращает ошибку шага sync с диагностикой, при этом факт успешной локальной нормализации сохраняется
