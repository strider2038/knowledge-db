# node-action-panel Specification

## Purpose

Панель действий на странице узла (Action Bar): toggle «Проверено», кнопка «Перевести», переключатель языков, «Переместить», «Удалить» с визуальным разделением групп.
## Requirements
### Requirement: Панель действий на странице узла

Страница узла MUST отображать панель действий (Action Bar) — горизонтальную полосу с кнопками, расположенную после breadcrumbs и перед заголовком узла. Панель MUST содержать слева направо: toggle «Проверено» (существующий функционал), кнопку «Перевести» (для статей без перевода), кнопку «Нормализация» (для запуска нормализации текущего узла), кнопку «Редактировать с агентом» (для оригинальных узлов), кнопку «Выгрузить изображения» (для article-узлов), кнопку «Обновить описание из источника» для узлов с `source_url`, разделитель, кнопку «Переместить», кнопку «Удалить». Деструктивные кнопки (Переместить, Удалить) MUST визуально отличаться (outline/ghost вместо default). Кнопки «Нормализация» и «Выгрузить изображения» MUST иметь состояние выполнения (loading/disabled) на время операции. Панель SHOULD быть sticky при прокрутке.

#### Сценарий: Отображение панели для article без перевода

- **WHEN** открыт узел типа article без переводов и с `source_url`
- **THEN** панель содержит: toggle «Проверено», кнопку «Перевести», кнопку «Нормализация», кнопку «Выгрузить изображения», кнопку «Обновить описание из источника», разделитель, «Переместить», «Удалить»

#### Сценарий: Отображение панели для article с переводами

- **WHEN** открыт узел типа article с переводами и с `source_url`
- **THEN** панель содержит: toggle «Проверено», переключатель языков, кнопку «Нормализация», кнопку «Выгрузить изображения», кнопку «Обновить описание из источника», разделитель, «Переместить», «Удалить» (без кнопки «Перевести»)

#### Сценарий: Отображение панели для article без source_url

- **WHEN** открыт узел типа article без `source_url`
- **THEN** панель содержит кнопки «Нормализация» и «Выгрузить изображения» и не содержит кнопку «Обновить описание из источника»

#### Сценарий: Блокировка повторного запуска нормализации

- **WHEN** пользователь нажал «Нормализация» и операция ещё выполняется
- **THEN** кнопка «Нормализация» disabled и показывает состояние выполнения до получения финального статуса

#### Сценарий: Ошибка нормализации

- **WHEN** операция нормализации завершилась ошибкой
- **THEN** UI показывает диагностируемое сообщение и возвращает кнопку «Нормализация» в обычное состояние

#### Сценарий: Блокировка повторного запуска dump images

- **WHEN** пользователь нажал «Выгрузить изображения» и операция ещё выполняется
- **THEN** кнопка «Выгрузить изображения» disabled и показывает состояние выполнения до получения финального статуса

#### Сценарий: Отображение панели для note/link с source_url

- **WHEN** открыт узел типа note или link с `source_url`
- **THEN** панель содержит: toggle «Проверено», кнопку «Обновить описание из источника», разделитель, «Переместить», «Удалить» (без кнопок «Перевести» и «Выгрузить изображения»)

#### Сценарий: Отображение панели для note/link без source_url

- **WHEN** открыт узел типа note или link без `source_url`
- **THEN** панель содержит: toggle «Проверено», разделитель, «Переместить», «Удалить» (без кнопок «Обновить описание из источника» и «Выгрузить изображения»)

#### Сценарий: Отображение панели для перевода

- **WHEN** открыт перевод узла (path содержит `.`)
- **THEN** панель содержит: переключатель языков, разделитель, «Переместить», «Удалить» (без toggle «Проверено», «Перевести», «Нормализация» и «Выгрузить изображения»)

### Requirement: Действие обновления описания из источника

UI ДОЛЖЕН (SHALL) предоставлять на странице узла действие «Обновить описание из источника» для узлов с `source_url`. При нажатии UI MUST вызвать `POST /api/nodes/{path}/refresh-description`, заблокировать повторное нажатие до завершения запроса и отобразить состояние выполнения. При успехе UI MUST обновить текущие данные узла без полного перехода на обзор и показать уведомление об успешном обновлении. При ошибке UI MUST показать диагностируемое сообщение.

#### Сценарий: Успешное обновление описания

- **WHEN** пользователь нажимает «Обновить описание из источника» и API возвращает обновлённый узел
- **THEN** UI обновляет заголовок, метаданные, аннотацию и тело текущей страницы согласно ответу API

#### Сценарий: Блокировка повторного запуска

- **WHEN** refresh-description запрос выполняется
- **THEN** кнопка «Обновить описание из источника» disabled и показывает loading-состояние

#### Сценарий: Ошибка обновления

- **WHEN** API возвращает ошибку при обновлении описания
- **THEN** UI показывает сообщение об ошибке и оставляет текущие данные узла без локального перезаписывания

#### Сценарий: Узел сменил type после обновления

- **WHEN** API возвращает узел с изменённым `type`, например `link` стал `note`
- **THEN** UI обновляет отображаемые метаданные и доступные действия панели согласно новому типу

### Requirement: Визуальное разделение групп действий

Панель действий MUST визуально разделять кнопки на группы: основные действия (Проверено, Перевести) и деструктивные/меняющие структуру (Переместить, Удалить). Между группами MUST быть вертикальный разделитель (divider). Кнопка «Удалить» MUST использовать destructive визуальный стиль (красный цвет при hover/active).

#### Сценарий: Визуальное разделение

- **WHEN** панель действий отображается с несколькими группами
- **THEN** между группами виден вертикальный разделитель, «Удалить» имеет красный hover-стиль

### Requirement: Нижняя разворачиваемая панель логов нормализации

UI SHALL отображать на странице узла отдельную нижнюю панель логов нормализации с режимами collapsed/expanded. Панель MUST показывать текущий статус операции (`running/success/error`) и список лог-записей в хронологическом порядке.

#### Сценарий: Открытие панели во время операции

- **WHEN** пользователь запускает «Нормализация»
- **THEN** UI показывает панель логов и начинает периодически обновлять статус и новые лог-записи

#### Сценарий: Ошибка операции

- **WHEN** операция завершилась ошибкой
- **THEN** панель показывает последние лог-записи и финальный error-статус

#### Сценарий: Свернуть/развернуть

- **WHEN** пользователь переключает панель
- **THEN** UI сохраняет и применяет выбранный режим отображения

### Requirement: Нижняя разворачиваемая панель логов dump images

UI SHALL отображать на странице узла отдельную нижнюю панель логов операции `dump images` с режимами `collapsed/expanded`. Панель MUST показывать текущий статус операции (`running/success/error`) и список лог-записей в хронологическом порядке.

#### Сценарий: Открытие панели во время операции
- **WHEN** пользователь запускает «Выгрузить изображения»
- **THEN** UI показывает панель логов и начинает периодически обновлять статус и новые лог-записи

#### Сценарий: Ошибка операции
- **WHEN** операция `dump images` завершилась ошибкой
- **THEN** панель показывает последние лог-записи и финальный error-статус

#### Сценарий: Свернуть/развернуть
- **WHEN** пользователь переключает панель
- **THEN** UI сохраняет и применяет выбранный режим отображения

### Requirement: Действие «Редактировать с агентом» на панели узла

UI SHALL предоставлять на панели действий страницы узла кнопку «Редактировать с агентом» для оригинальных узлов (не для переводов). Кнопка MUST быть доступна для типов `article`, `note` и `link` наравне с «Нормализация». Во время выполнения операции кнопка MUST быть disabled и показывать состояние выполнения.

#### Scenario: Отображение кнопки на оригинальном узле

- **WHEN** открыт оригинальный узел (не перевод)
- **THEN** панель содержит кнопку «Редактировать с агентом»

#### Scenario: Скрытие на переводе

- **WHEN** открыт перевод узла (path перевода)
- **THEN** кнопка «Редактировать с агентом» не отображается

#### Scenario: Блокировка повторного запуска

- **WHEN** операция agent-edit выполняется
- **THEN** кнопка disabled до финального статуса операции

### Requirement: Модальное окно инструкции для агента

UI SHALL открывать модальное окно при нажатии «Редактировать с агентом». Модалка MUST содержать многострочное поле для текстовой инструкции, краткое пояснение назначения и кнопки «Отмена» и «Запустить». «Запустить» MUST быть disabled, пока инструкция пуста (после trim). При успешном старте модалка MUST закрываться и запускаться операция на сервере.

#### Scenario: Отмена без запуска

- **WHEN** пользователь нажимает «Отмена» или закрывает модалку
- **THEN** операция не запускается, состояние страницы не меняется

#### Scenario: Запуск с инструкцией

- **WHEN** пользователь вводит инструкцию и нажимает «Запустить»
- **THEN** UI вызывает API agent-edit с телом `instruction` и переходит к отображению прогресса

#### Scenario: Ошибка старта

- **WHEN** API возвращает ошибку при запуске
- **THEN** UI показывает диагностируемое сообщение; модалка MAY оставаться открытой для исправления инструкции

### Requirement: Сохранение последней инструкции в sessionStorage

UI SHALL сохранять в `sessionStorage` последнюю инструкцию, успешно отправленную при запуске agent-edit, с ключом, привязанным к `node_path` текущего узла. При открытии модалки UI MUST подставлять сохранённую инструкцию в поле ввода, если значение для этого path существует. UI MUST NOT сохранять инструкцию при отмене модалки без запуска.

#### Scenario: Повторное открытие модалки

- **WHEN** пользователь ранее успешно запускал agent-edit для этого узла в текущей вкладке
- **THEN** при открытии модалки поле инструкции содержит последний сохранённый текст

#### Scenario: Успешный запуск обновляет сохранение

- **WHEN** API принял запуск agent-edit с инструкцией
- **THEN** UI записывает эту инструкцию в `sessionStorage` для path узла

#### Scenario: Отмена без сохранения

- **WHEN** пользователь закрыл модалку без успешного запуска
- **THEN** ранее сохранённая инструкция для path не перезаписывается введённым черновиком

### Requirement: Нижняя разворачиваемая панель логов agent-edit

UI SHALL отображать на странице узла отдельную нижнюю панель логов операции agent-edit с режимами collapsed/expanded. Панель MUST показывать текущий статус (`running/success/error`), стадию и список лог-записей в хронологическом порядке с периодическим polling статуса и логов (как у нормализации). Панель MUST поддерживать режимы отображения compact/raw. При запуске agent-edit UI SHOULD сворачивать панель логов нормализации и наоборот.

#### Scenario: Открытие панели при запуске

- **WHEN** пользователь успешно запустил agent-edit
- **THEN** UI показывает панель логов и начинает обновлять статус и новые записи

#### Scenario: Успешное завершение

- **WHEN** операция завершилась success
- **THEN** панель показывает финальный статус; UI обновляет данные узла на странице

#### Scenario: Ошибка операции

- **WHEN** операция завершилась error
- **THEN** панель показывает последние лог-записи и сообщение об ошибке
