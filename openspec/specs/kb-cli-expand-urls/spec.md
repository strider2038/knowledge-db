## Purpose

Зафиксировать поведение подкоманды kb-cli `expand-urls`: раскрытие редиректных коротких ссылок до канонического `https`-адреса и удаление UTM/трекинговых query-параметров в содержимом markdown-файлов базы знаний.

## Requirements

### Requirement: Нормализация URL

Система SHALL для каждого обнаруженного в обрабатываемом файле URL со схемой `http` или `https` вызывать ту же нормализацию, что и `internal/pkg/urlutil.NormalizeURL`: следование HTTP-редиректам (раскрытие сокращений), удаление параметров запроса с префиксом `utm_`, а также иные трекинг-параметры, если они реализованы в общей функции нормализации. Для хостов из списка исключений в `urlutil` (например `t.me`, `telegram.org`) поведение MUST совпадать с серверной логикой (без раскрытия редиректа как целевого URL).

#### Scenario: Раскрытие редиректа

- **WHEN** в markdown есть ссылка `https://short.example/x`, сервер отвечает редиректом на полный URL
- **THEN** в файле после успешного запуска команды URL MUST быть заменён на конечный адрес (без лишних UTM, если они снимаются нормализатором)

#### Scenario: Сеть недоступна

- **WHEN** HTTP-запрос нормализации завершается ошибкой или таймаутом
- **THEN** исходная строка URL в файле MUST остаться без изменений (как в `NormalizeURL`)

### Requirement: Область замены в markdown

Система SHALL заменять URL как минимум внутри markdown-ссылок вида `[текст](URL)` и в угловых автоссылках `<URL>`. Система SHOULD нормализовать URL в значениях полей frontmatter (YAML), если значение представляет собой один URL в кавычках.

#### Scenario: Ссылка в теле статьи

- **WHEN** в теле есть `[Репозиторий](https://clc.to/abc)`
- **THEN** после обработки href MUST быть заменён на результат нормализации

### Requirement: Флаги CLI

Подкоманда SHALL поддерживать `--path` (корень базы), обязательный `--file` (путь к `.md` относительно корня или абсолютный под корнем базы), `--dry-run` (только вывести пары старый/новый URL без записи файла).

#### Scenario: Dry-run

- **WHEN** пользователь запускает `kb-cli expand-urls --path . --file x.md --dry-run`
- **THEN** файл на диске не изменяется, в выводе отображаются предполагаемые замены

### Requirement: Отчёт об ошибках

Если для части URL нормализация не удалась (сеть, таймаут), команда SHALL применить замены для успешно нормализованных URL, вывести предупреждение на stderr по каждому неудачному URL и завершиться с ненулевым кодом выхода, если была хотя бы одна ошибка нормализации.

#### Scenario: Частичный успех

- **WHEN** один из нескольких URL не удалось нормализовать
- **THEN** остальные замены применяются, по проблемному URL выводится предупреждение, код выхода ненулевой
