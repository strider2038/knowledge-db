## Purpose

Автоматический перевод статей (type=article) на русский при сохранении. Определение языка по эвристике, вызов LLM для перевода, чанкинг для длинных текстов.

## Requirements

### Requirement: Определение необходимости перевода

Система ДОЛЖНА (SHALL) определять, нужен ли перевод контента на русский, по комбинированной эвристике: удаление code blocks из текста, подсчёт доли кириллицы среди букв. Если `cyrillic_ratio < 0.25` — контент считается на английском и требует перевода. Минимальная длина для оценки — 200 символов (после удаления code blocks); при меньшей длине перевод не выполняется.

#### Сценарий: Контент преимущественно на английском

- **WHEN** текст после удаления code blocks содержит менее 25% кириллицы и более 200 символов
- **THEN** система определяет, что перевод нужен

#### Сценарий: Контент преимущественно на русском

- **WHEN** текст содержит более 25% кириллицы
- **THEN** система определяет, что перевод не нужен

#### Сценарий: Текст слишком короткий

- **WHEN** после удаления code blocks осталось менее 200 символов
- **THEN** система не выполняет перевод

### Requirement: Перевод через LLM

Система ДОЛЖНА (SHALL) переводить контент на русский через OpenAI Responses API (без tools). Input — текст для перевода, Instructions — сохранение структуры markdown (заголовки, списки, code blocks без изменений). Ответ — переведённый текст. Система MUST использовать ту же модель и клиент, что и для ingestion orchestrator.

#### Сценарий: Успешный перевод короткого текста

- **WHEN** контент менее 6000 символов и вызывается перевод
- **THEN** LLM возвращает полный перевод за один вызов

#### Сценарий: Сохранение структуры markdown

- **WHEN** исходный текст содержит заголовки, списки, code blocks
- **THEN** перевод сохраняет эту структуру; code blocks остаются без изменений

### Requirement: Чанкинг длинных статей

Система ДОЛЖНА (SHALL) разбивать контент на чанки при длине более 6000 символов (после удаления code blocks). Разбиение — по границам абзацев; целевой размер чанка — ~4000 символов. Code blocks вырезаются перед разбиением и вставляются обратно после склейки. Каждый чанк переводится отдельным вызовом LLM; для чанков 2+ в промпт добавляется контекст конца предыдущей части. При склейке система MUST проверять и удалять дублирование на стыках.

#### Сценарий: Перевод длинной статьи

- **WHEN** контент более 6000 символов
- **THEN** система разбивает на чанки, переводит по частям, склеивает результат

#### Сценарий: Code blocks не переводятся

- **WHEN** в тексте есть code blocks
- **THEN** при чанкинге они извлекаются, не переводятся и вставляются обратно в итоговый текст
