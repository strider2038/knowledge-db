# node-move Specification

## Purpose

Перемещение узла через REST API и UI: API-эндпоинт POST …/move, модалка с текстовым полем полного пути и деревом тем, рекурсивное создание промежуточных директорий.
## Requirements
### Requirement: API перемещения узла

REST API MUST предоставлять эндпоинт `POST /api/nodes/{path}/move` для перемещения узла. Тело запроса MUST содержать `target_path` (полный целевой путь: `{тема/.../тема}/{slug}`). При успехе MUST возвращаться обновлённый объект узла (как GET) со статусом 200; объект MUST содержать тот же `id`, что до перемещения, и новый `path`. Если узел не найден — 404. Если целевой путь занят — 409 с сообщением о конфликте. Перемещение MUST переносить главный `.md` файл и директорию вложений. Промежуточные директории целевого пути MUST создаваться рекурсивно при необходимости. Поле `id` во frontmatter MUST NOT изменяться.

#### Сценарий: Успешное перемещение в другую тему

- **WHEN** `POST /api/nodes/old-topic/my-node/move` с `{ "target_path": "new-topic/my-node" }`
- **THEN** файл перемещается, возвращается объект с `path`=`new-topic/my-node` и неизменным `id`

#### Сценарий: Перемещение с изменением slug

- **WHEN** `POST /api/nodes/topic/old-name/move` с `{ "target_path": "topic/new-name" }`
- **THEN** файл переименован, `id` в frontmatter и в JSON-ответе совпадает с id до move

#### Сценарий: Целевой путь занят

- **WHEN** `POST /api/nodes/topic/my-node/move` с `{ "target_path": "other/my-node" }` и в `other/` уже существует `my-node.md`
- **THEN** возвращается 409 с сообщением "target path already exists"

#### Сценарий: Узел не найден

- **WHEN** `POST /api/nodes/nonexistent/move` с `{ "target_path": "target/node" }`
- **THEN** возвращается 404

#### Сценарий: Рекурсивное создание промежуточных директорий

- **WHEN** `POST /api/nodes/topic/node/move` с `{ "target_path": "new/deep/path/node" }` и директории `new/`, `new/deep/`, `new/deep/path/` не существуют
- **THEN** создаются все промежуточные директории, узел перемещается, `id` сохраняется

#### Сценарий: Пустой target_path

- **WHEN** `POST /api/nodes/topic/node/move` с `{ "target_path": "" }` или без поля target_path
- **THEN** возвращается 400 с сообщением "target_path is required"

#### Сценарий: Некорректный target_path

- **WHEN** `POST /api/nodes/topic/node/move` с `{ "target_path": "../etc/passwd" }` (path traversal)
- **THEN** возвращается 400 с сообщением "invalid target_path"

### Requirement: Модалка перемещения узла

UI MUST предоставлять модалку для перемещения узла. Модалка MUST содержать: текущий путь (нередактируемый, для справки), редактируемое текстовое поле «Новый путь» с предзаполненным текущим путём, дерево существующих тем (клик по теме обновляет текстовое поле). Текстовое поле «Новый путь» MUST позволять вводить произвольный полный путь, включая несуществующие темы — промежуточные директории создаются автоматически при подтверждении. Кнопка «Переместить» MUST быть активна только если путь фактически изменён и нет конфликта.

#### Сценарий: Открытие модалки

- **WHEN** пользователь нажимает «Переместить» на панели действий
- **THEN** открывается модалка с текущим путём, текстовым полем «Новый путь» (предзаполнено текущим путём), деревом тем (раскрытым на текущей теме)

#### Сценарий: Выбор темы в дереве

- **WHEN** пользователь выбирает тему в дереве
- **THEN** текстовое поле «Новый путь» обновляется: `{выбранная_тема}/{текущий_slug}`

#### Сценарий: Прямое редактирование полного пути

- **WHEN** пользователь редактирует текстовое поле «Новый путь» напрямую
- **THEN** предпросмотр обновляется, путь может содержать несуществующие темы

#### Сценарий: Перемещение без изменений

- **WHEN** «Новый путь» совпадает с текущим
- **THEN** кнопка «Переместить» неактивна

#### Сценарий: Успешное перемещение с созданием новых тем

- **WHEN** пользователь вводит путь `new/topic/my-node` (директории не существуют) и нажимает «Переместить»
- **THEN** API создаёт промежуточные директории, узел перемещается, происходит переход на новый путь

#### Сценарий: Конфликт при перемещении

- **WHEN** API возвращает 409 (целевой путь занят)
- **THEN** отображается ошибка в модалке, модалка остаётся открытой

#### Сценарий: Индикация процесса перемещения

- **WHEN** запрос на перемещение выполняется
- **THEN** кнопка «Переместить» disabled, отображается спиннер
