## Context

На странице «Обзор» дерево тем строится из `getTree()` и рендерится рекурсивно в `OverviewPage`. Сейчас все ветки развёрнуты полностью. URL уже хранит `path` выбранной темы.

## Goals / Non-Goals

**Goals:**

- Раскрытие/сворачивание веток без потери выбора темы кликом по названию.
- Настраиваемая глубина «автораскрытия» (по умолчанию 2 уровня пути темы).
- Обязательное раскрытие цепочки предков до текущего `path` из URL.
- Сохранение настройки и ручных раскрытий/сворачиваний в `localStorage`.

**Non-Goals:**

- Синхронизация состояния дерева между устройствами или с сервером.
- Изменение REST API и формата дерева на backend.

## Decisions

1. **Разделение «выбор темы» и «раскрыть ветку»**  
   Клик по **названию темы** по-прежнему задаёт `path` в query (как сейчас). Рядом с узлами, у которых есть дочерние темы в дереве, — **кнопка chevron**: только она переключает раскрытие (`stopPropagation`), чтобы не ломать сценарий выбора.

2. **Вычисление «обязательно раскрыть»**  
   Для `?path=a/b/c` в множество принудительно раскрытых входят **все строгие префиксы** пути (`a`, `a/b`), чтобы выбранная тема была видна в списке. Сам лист `a/b/c` не обязан быть «раскрыт» (детей нет или не требуется показывать).

3. **Глубина по умолчанию**  
   Узел с путём `p` считается имеющим «глубину» = число сегментов `p.split('/').length`. Если глубина ≤ настройки `defaultExpandDepth` (по умолчанию 2), ветка считается раскрытой по умолчанию, пока пользователь явно не свернул её (см. ниже).

4. **Ручные переопределения**  
   В `localStorage` хранится объект: `defaultExpandDepth`, массивы `userExpanded` и `userCollapsed` (строковые пути).  
   Итог: `expanded(p) = mustExpand(p) || ((!userCollapsed.has(p)) && (defaultByDepth(p) || userExpanded.has(p)))`, где `mustExpand` — предки текущего `path` из URL; для узлов на обязательном пути `userCollapsed` игнорируется.

5. **Версионирование ключа**  
   Ключ вида `kb.overview.topicNav.v1`, чтобы при смене схемы не ломать старые клиенты без миграции.

## Risks / Trade-offs

- **[Risk]** Переполнение `localStorage` при очень частых ручных раскрытиях → **Mitigation:** хранить только пути строк; при необходимости в будущем — ограничить размер или сброс.
- **[Trade-off]** Состояние дерева не в URL — при шаринге ссылки виден только `path`, не пользовательские раскрытия; это приемлемо для локального инструмента.

## Migration Plan

Только деплой фронтенда; откат — откат сборки web. Старые клиенты без ключа получают дефолты.
