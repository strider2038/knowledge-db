## Purpose

Дополнение к спецификации обзора: раскрываемое дерево тем с настройкой глубины и локальным сохранением состояния.

## Requirements

## ADDED Requirements

### Requirement: Раскрываемое дерево тем

UI ДОЛЖЕН (SHALL) отображать узлы дерева тем с возможностью сворачивать и разворачивать ветки, если у темы есть дочерние темы в дереве. Выбор темы (фильтр таблицы по path) MUST оставаться отдельным действием от переключения раскрытия ветки.

#### Сценарий: Раздельное действие

- **WHEN** у темы есть дочерние узлы в дереве
- **THEN** пользователь MAY свернуть или развернуть ветку без смены выбранной темы, если для этого предусмотрен отдельный контроль (например, chevron)

### Requirement: Глубина раскрытия по умолчанию

Система SHALL поддерживать настройку «сколько уровней вложенности тем раскрывать по умолчанию» (целое число ≥ 1). Значение по умолчанию MUST быть 2. Узел с путём `p` считается уровнем `segments(p)`, где `segments` — число сегментов пути, разделённых `/`.

#### Сценарий: Настройка в интерфейсе

- **WHEN** пользователь меняет число уровней по умолчанию
- **THEN** новое значение применяется к отображению дерева и сохраняется локально в браузере

### Requirement: Путь до текущей темы

При активном query-параметре `path` все предки выбранной темы в дереве MUST быть раскрыты так, чтобы выбранная тема была видна в списке (цепочка от корня до выбранного узла).

#### Сценарий: Загрузка с глубоким path

- **WHEN** пользователь открывает обзор с `?path=a/b/c` и дерево содержит эту ветку
- **THEN** узлы `a` и `a/b` отображаются развёрнутыми независимо от настройки глубины по умолчанию

### Requirement: Сохранение состояния дерева

Состояние раскрытия веток (и настройка глубины по умолчанию) MUST сохраняться в браузере (например, `localStorage`) и восстанавливаться после перезагрузки страницы, не конфликтуя с восстановлением `path` из URL.

#### Сценарий: Перезагрузка

- **WHEN** пользователь перезагружает страницу обзора
- **THEN** настройка глубины и ручные раскрытия/сворачивания учитываются снова, а `path` из URL по-прежнему определяет выбранную тему и принудительно раскрывает путь до неё
