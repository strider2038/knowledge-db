# Proposal: Автоматический перевод статей на русский

## Why

Многие сохраняемые статьи приходят на английском. Автоматический перевод при сохранении позволит пользователю читать материалы на родном языке без ручной работы. База знаний становится доступнее при сохранении принципа оффлайн-first (перевод через LLM, без внешних API).

## What Changes

- Автоматический перевод статей (type=article) на русский при сохранении, если контент преимущественно на английском
- Хранение перевода в отдельном файле `{slug}.ru.md` рядом с оригиналом
- Связь оригинала и перевода через frontmatter (`translation_of`, `translations`) и wikilinks для совместимости с Obsidian
- Дублирование метаданных (keywords, annotation, title и т.д.) в файл перевода
- Определение языка по комбинированной эвристике: исключение code blocks + доля кириллицы в тексте
- Перевод выполняется только для type=article; обновление перевода при изменении оригинала не предусмотрено (ответственность пользователя)

## Capabilities

### New Capabilities

- `article-translation`: Определение языка контента (эвристика), вызов LLM для перевода на русский, сохранение перевода. Охватывает логику «нужен ли перевод» и «как перевести».

### Modified Capabilities

- `knowledge-storage`: Добавление формата хранения переводов — файл `{slug}.ru.md` в той же теме, frontmatter с полями `translation_of`, `lang`, `translations`; wikilinks в теле для связи с оригиналом
- `ingestion-pipeline`: Добавление пост-шага после создания узла: для type=article при срабатывании эвристики «контент на английском» — вызов перевода и сохранение файла перевода
- `kb-cli`: Расширение validate — проверка файлов переводов (`*.ru.md`): frontmatter (`translation_of`, `lang`), валидация wikilinks (ссылки должны вести на существующие узлы)

## Impact

- **internal/ingestion**: новый пакет или расширение pipeline — Translator/TranslationStep, интеграция в saveNode
- **internal/kb**: расширение Store (CreateTranslationNode или аналог) для записи `{slug}.ru.md`; утилита определения языка; расширение Validate для проверки переводов и wikilinks
- **internal/ingestion/llm**: дополнительный вызов LLM для перевода (Responses API, без tools)
- **cmd/kb-cli**: расширение validate — проверка `*.ru.md`, валидация ссылок
- **Конфигурация**: env `KB_AUTO_TRANSLATE` (default true)
