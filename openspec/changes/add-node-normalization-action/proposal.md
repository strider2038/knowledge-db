## Why

Сейчас на странице узла нет отдельного действия для целевой нормализации содержимого через Cursor Agent, из-за чего нормализация выполняется вручную и непоследовательно. Нужен управляемый запуск из UI с гарантированным `sync` после успешного завершения, чтобы результат сразу попадал в git-first workflow.

## What Changes

- Добавить в Action Bar страницы узла новое действие «Нормализация».
- Добавить backend endpoint для запуска Cursor Agent с промтом, сформированным из контекста текущего узла.
- Определить контракт асинхронного выполнения: старт задачи, статус, завершение, ошибка.
- После успешного завершения нормализации автоматически запускать `sync` для фиксации и распространения изменений в локальном git-процессе.
- Обновить runtime-окружение: установить Cursor Agent в Docker-образ.

## Capabilities

### New Capabilities
- `node-normalization`: Нормализация отдельного узла через Cursor Agent с последующим sync.

### Modified Capabilities
- `node-action-panel`: Добавление кнопки «Нормализация» в панель действий узла и состояний выполнения.
- `rest-api`: Добавление endpoint(ов) запуска/статуса нормализации и серверной оркестрации sync после успеха.
- `kb-unified-binary`: Расширение runtime-конфигурации окружения для работы Cursor Agent в self-hosted/Docker режиме.

## Impact

- `web/`: UI страницы узла и API-клиент для действия нормализации.
- `internal/api`, `internal/bootstrap` (и/или связанный runtime слой): запуск Cursor Agent, обработка статусов, запуск sync.
- Docker-артефакты: установка Cursor Agent через `curl https://cursor.com/install -fsS | bash`.
- Runtime-зависимость: доступный `cursor-agent` в серверном окружении (документация и контейнерный образ).
