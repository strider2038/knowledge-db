# Proposal: Обзор базы знаний — улучшенный UI

## Why

Текущий раздел «Поиск» показывает плоский список узлов выбранной темы без фильтрации, без метаданных (тип, дата, ссылка) и без учёта переводов. Пользователю сложно ориентироваться в базе, находить нужный тип контента (статьи, ссылки, заметки) и быстро переходить к источникам. Переименование в «Обзор» и доработка UI решают задачу удобного просмотра и навигации по всей базе знаний.

## What Changes

### Frontend (webapp)

- Переименовать пункт меню «Поиск» → «Обзор»
- Древовидная структура слева — остаётся, служит фильтром по поддереву
- В правой части — список узлов **всего поддерева** (рекурсивно), а не только выбранной папки
- Фильтры по типам узлов: toggle-кнопки article / link / note; дерево и список фильтруются — пустые ветки скрываются
- Сортируемая таблица с колонками: Название (из title), Тип (цветом), Дата, Ссылка (для link/article — кликабельная, target="_blank")
- Схлопывание переводов: article.ru.md не показывается отдельно; на странице узла — выбор языка при наличии переводов
- Инпут поиска вверху: фильтрация по подстроке в заголовке, ключевиках, аннотации (серверный поиск)
- Breadcrumbs в дереве при выборе темы
- Вариант «Вся база» в дереве — показ узлов из всего дерева
- Сортировка колонок с сохранением в URL
- Сохранение фильтров (типы, поиск) в URL
- Пагинация таблицы

### Backend (rest-api, internal/kb)

- Новый API: рекурсивный список узлов по поддереву — `GET /api/nodes?path=X&recursive=true` (или аналог)
- Расширение ответа списка узлов: метаданные `type`, `title`, `created`, `source_url`, `keywords`, `translations` — для отображения в таблице без отдельного запроса на каждый узел
- Схлопывание переводов на стороне API: в списке — только оригиналы, переводы в поле `translations`
- Поиск по тексту — параметр `q` в `GET /api/nodes`: `?path=X&recursive=true&q=...` фильтрует по вхождению подстроки в title, keywords, annotation

## Capabilities

### New Capabilities

- `overview-browse`: страница «Обзор» — дерево-фильтр, таблица узлов поддерева, фильтры по типам, поиск, сортировка, пагинация, схлопывание переводов

### Modified Capabilities

- `webapp`: раздел «Поиск» заменяется на «Обзор» с новой логикой (требования к Navbar, структуре страницы, таблице)
- `rest-api`: расширение `GET /api/nodes` — параметры path, recursive, q; расширенный ответ с метаданными

## Impact

- **internal/kb**: расширение `ListNodes` или новый метод для рекурсивного списка с метаданными; логика определения оригинала vs перевода; реализация поиска по title/keywords/annotation
- **internal/api**: новые query-параметры для `/api/nodes` (path, recursive, q)
- **web/src**: SearchPage → OverviewPage (или рефакторинг), новые компоненты (фильтры, таблица с сортировкой, breadcrumbs, пагинация)
- **web/src/components/Navbar**: переименование «Поиск» → «Обзор»
- **web/src/pages/NodePage**: выбор языка при наличии переводов
- API-тесты для новых/изменённых эндпоинтов
