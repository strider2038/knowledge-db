## ADDED Requirements

### Requirement: Вкладка гибридного поиска

UI ДОЛЖЕН (SHALL) отображать вкладку «Поиск» в Navbar при доступности гибридного индекса. Вкладка MUST содержать поле ввода запроса, фильтры по типу ноды, теме/поддереву и ручной проверке, а также список результатов в виде карточек. Каждая карточка MUST показывать title, type, annotation, path, keywords, причины совпадения и релевантные фрагменты, если они есть. При недоступном индексе вкладка MUST быть скрыта или показывать понятное состояние недоступности.

#### Scenario: Поиск по базе

- **WHEN** пользователь вводит запрос на вкладке «Поиск»
- **THEN** UI отправляет `POST /api/search` и отображает карточки результатов

#### Scenario: Карточка article с фрагментом

- **WHEN** API возвращает result с fragments
- **THEN** карточка показывает heading/snippet релевантного фрагмента

#### Scenario: Нет результатов

- **WHEN** API возвращает пустой список results
- **THEN** UI отображает пустое состояние с текстом, что ничего не найдено

#### Scenario: Индекс недоступен

- **WHEN** `GET /api/index/status` возвращает 503
- **THEN** вкладка «Поиск» не отображается в Navbar или показывает состояние недоступности без падения приложения

### Requirement: Переход из поиска в чат

UI ДОЛЖЕН (SHALL) позволять пользователю перейти из результатов поиска в чат с сохранением запроса и выбранных источников. Карточка результата или группа результатов MUST иметь действие “Спросить по этим источникам” или эквивалентное действие, которое открывает вкладку «Чат» с initial message/source hints.

#### Scenario: Спросить по найденным источникам

- **WHEN** пользователь нажимает действие “Спросить по этим источникам” на странице поиска
- **THEN** UI открывает чат с предзаполненным вопросом и передаёт выбранные source_paths в `POST /api/chat`

#### Scenario: Открыть найденную ноду

- **WHEN** пользователь нажимает на title карточки результата
- **THEN** UI открывает страницу соответствующей ноды

## MODIFIED Requirements

### Requirement: Вкладка чатбота

UI ДОЛЖЕН (SHALL) отображать вкладку «Чат» в Navbar при `KB_EMBEDDING_ENABLED=true` и доступности RAG-индекса. Вкладка MUST содержать: поле ввода сообщения, область ответа (streaming), список источников (path + title + type, кликабельные), и возможность раскрыть найденные фрагменты контекста при наличии. При `KB_EMBEDDING_ENABLED=false` вкладка MUST быть скрыта. UI MUST определять доступность чата через `GET /api/index/status` — при 503 вкладка скрывается.

#### Scenario: Отправка сообщения чатботу

- **WHEN** пользователь вводит сообщение и нажимает отправку
- **THEN** сообщение отправляется `POST /api/chat`, ответ отображается streaming, источники отображаются до или рядом с ответом

#### Scenario: Чатбот недоступен

- **WHEN** `GET /api/index/status` возвращает 503
- **THEN** вкладка «Чат» не отображается в Navbar

#### Scenario: Отображение источников

- **WHEN** чатбот возвращает ответ с sources
- **THEN** UI отображает кликабельные ссылки на ноды базы знаний с path, title, type и фрагментами, если они есть

#### Scenario: Недостаточно данных в базе

- **WHEN** чат возвращает пустые sources или ответ о недостатке контекста
- **THEN** UI отображает это как нормальное состояние, а не как техническую ошибку
