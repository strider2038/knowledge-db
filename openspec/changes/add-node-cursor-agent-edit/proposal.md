## Why

Редактирование узла через Cursor Agent сейчас возможно только косвенно (фиксированная «Нормализация»). Пользователю нужен произвольный сценарий правки узла: описать задачу текстом, запустить агента с сервера и видеть ход выполнения в UI, не уходя со страницы узла.

## What Changes

- Добавить действие «Редактировать с агентом» на панели страницы узла (для оригинальных узлов, не для переводов).
- Добавить модальное окно с многострочным полем инструкции для агента и подтверждением запуска.
- Добавить серверный запуск `cursor-agent` с промтом на основе инструкции пользователя и контекста текущего узла.
- Добавить асинхронную операцию со статусом, стадиями (`edit` → `sync` → `done`) и потоковыми логами (как у нормализации).
- Добавить нижнюю разворачиваемую панель логов операции редактирования на странице узла.
- После успешного редактирования автоматически выполнять `sync` (git-first поток).

## Capabilities

### New Capabilities

- `node-cursor-agent-edit`: Серверный сценарий произвольного редактирования узла через Cursor Agent по пользовательской инструкции, сбор логов и post-sync.

### Modified Capabilities

- `rest-api`: Endpoint-ы запуска, статуса и логов операции agent-edit.
- `node-action-panel`: Кнопка, модалка инструкции, панель логов и состояния выполнения на странице узла.

## Impact

- Backend `internal/api`: runner для `cursor-agent`, job type, handlers, переиспользование парсинга stream-json и job store.
- Frontend `web/src`: кнопка и модалка в `NodeActionBar`, API client, polling статуса/логов, панель логов.
- Тесты: API tests для новых endpoint-ов; component/page tests для UI сценариев.
- Runtime: требуется `cursor-agent` в PATH (как для нормализации).
