## Why

Сейчас действие «Нормализация» непрозрачно: пользователь видит только итоговый статус и не понимает, что происходит во время работы Cursor Agent. Нужен потоковый/инкрементальный лог и удобное отображение прямо на странице узла.

## What Changes

- Добавить сбор логов выполнения нормализации узла (stdout/stderr + системные события стадий).
- Добавить API получения логов операции нормализации с поддержкой инкрементального чтения (offset/after).
- Добавить в UI страницы узла разворачиваемую панель внизу страницы для просмотра логов.
- Добавить управление отображением логов: свернуть/развернуть, автообновление во время running, фильтрация по типу потока.

## Capabilities

### New Capabilities
- `node-normalization-logs`: Хранение и выдача логов операции нормализации узла для UI.

### Modified Capabilities
- `rest-api`: Расширение контрактов нормализации endpoint-ом логов операции.
- `node-action-panel`: Добавление нижней разворачиваемой панели логов нормализации на странице узла.

## Impact

- Backend `internal/api`: захват stdout/stderr `cursor-agent`, буферизация логов, endpoint `/api/node-normalization/{id}/logs`.
- Frontend `web/src`: UI-панель логов на NodePage, polling логов и статуса.
- Тесты backend/frontend для новых сценариев.
