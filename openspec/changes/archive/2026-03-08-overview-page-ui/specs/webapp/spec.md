## MODIFIED Requirements

### Requirement: Navbar с меню

UI ДОЛЖЕН (SHALL) содержать верхнюю панель навигации (Navbar) с пунктами меню: «Добавить», «Обзор».

#### Сценарий: Переключение между разделами

- **WHEN** пользователь выбирает пункт «Добавить» или «Обзор»
- **THEN** отображается соответствующий раздел

### Requirement: Раздел «Обзор»

При выборе «Обзор» MUST отображаться страница обзора базы: слева — иерархическое дерево тем (фильтр по поддереву), справа — таблица узлов с колонками Название, Тип, Дата, Ссылка. Дерево поддерживает «Вся база»; таблица — фильтры по типам (article/link/note), поиск по тексту, сортировку, пагинацию. Детали — в спецификации overview-browse.

#### Сценарий: Навигация по дереву

- **WHEN** пользователь раскрывает/выбирает тему в дереве
- **THEN** таблица справа обновляется и показывает узлы выбранного поддерева (рекурсивно)

#### Сценарий: Отображение списка узлов

- **WHEN** выбрана тема, подтема или «Вся база»
- **THEN** в таблице отображаются узлы с title, type, path, датой, ссылкой (для link/article)

### Requirement: Просмотр узла

UI ДОЛЖЕН (SHALL) позволять просматривать содержимое узла (annotation, content, metadata). При наличии переводов MUST отображаться выбор языка (оригинал / переводы).

#### Сценарий: Открытие узла

- **WHEN** пользователь выбирает узел из таблицы
- **THEN** отображается annotation, content, metadata

#### Сценарий: Выбор языка

- **WHEN** у узла есть переводы (translations)
- **THEN** отображается переключатель для выбора оригинала или перевода
