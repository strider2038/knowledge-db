## ADDED Requirements

### Requirement: Ручное редактирование заголовка и ключевых слов на странице узла

UI SHALL предоставлять на странице узла ручное редактирование `title` и `keywords` через модальные окна. Рядом с заголовком и рядом с блоком ключевых слов MUST отображаться иконка редактирования (карандаш). Нажатие по иконке MUST открывать соответствующую модалку и позволять сохранить изменения через `PATCH /api/nodes/{path...}` без перезагрузки страницы.

#### Сценарий: Редактирование заголовка

- **WHEN** пользователь нажимает карандаш у заголовка, вводит новое значение и нажимает «Сохранить»
- **THEN** UI отправляет PATCH с полем `title`, обновляет заголовок на текущей странице и закрывает модалку

#### Сценарий: Ошибка сохранения заголовка

- **WHEN** PATCH для заголовка возвращает ошибку
- **THEN** UI показывает диагностируемое сообщение и оставляет модалку открытой

### Requirement: UX редактирования тегов с подсказками

Модалка ключевых слов MUST использовать token-input UX: существующие keywords отображаются как chips с удалением, новые keywords можно добавлять через кнопку и с клавиатуры (`Enter` или `,`), удаление последнего тега через `Backspace` при пустом input. UI MUST показывать suggestions на основе уже существующих keywords в базе и позволять добавлять их кликом.

#### Сценарий: Добавление тега из suggestions

- **WHEN** пользователь открывает модалку ключевых слов и выбирает подсказку
- **THEN** выбранный keyword добавляется в список тегов, дубль не создаётся

#### Сценарий: Сохранение ключевых слов

- **WHEN** пользователь сохраняет модалку keywords
- **THEN** UI отправляет PATCH с массивом `keywords`, обновляет метаданные на странице узла и закрывает модалку

#### Сценарий: Пустой список ключевых слов

- **WHEN** у узла нет keywords
- **THEN** header страницы показывает состояние «без ключевых слов», но карандаш редактирования остаётся доступным
