## MODIFIED Requirements

### Requirement: Navbar с меню

UI ДОЛЖЕН (SHALL) содержать верхнюю панель навигации (Navbar) с пунктами меню: «Добавить», «Обзор». После успешного ответа `GET /api/git/status` Navbar MUST показывать справа (перед logout и theme toggle) элемент сохранения в git: при **включённом** git и **наличии** незакоммиченных изменений — активную кнопку **«Сохранить (N)»**; при включённом git и **отсутствии** изменений — неактивную кнопку **«Сохранить»** с подсказкой по наведению; при **отключённом** git (ответ 503) — всегда неактивную кнопку **«⚠️ Сохранить»** с подсказкой по наведению (причина: git отключён на сервере, например KB_GIT_DISABLED). UI MUST периодически (каждые 30 секунд) проверять git-статус. При нажатии активной кнопки выполняется коммит+пуш с LLM-генерацией сообщения. Результат (успех или ошибка) MUST показываться **всплывающим уведомлением снизу экрана** (toast), а не строкой рядом с кнопкой.

#### Сценарий: Переключение между разделами

- **WHEN** пользователь выбирает пункт «Добавить» или «Обзор»
- **THEN** отображается соответствующий раздел

#### Сценарий: Кнопка «Сохранить» активна

- **WHEN** GET /api/git/status возвращает has_changes: true (git не отключён)
- **THEN** в Navbar отображается активная кнопка «Сохранить» с количеством изменённых файлов

#### Сценарий: Кнопка «Сохранить» неактивна без изменений

- **WHEN** GET /api/git/status возвращает has_changes: false и git не отключён
- **THEN** в Navbar отображается неактивная кнопка «Сохранить» с подсказкой по наведению

#### Сценарий: Git отключён на сервере

- **WHEN** GET /api/git/status возвращает 503
- **THEN** в Navbar отображается неактивная кнопка «⚠️ Сохранить» с подсказкой по наведению о причине

#### Сценарий: Успешное сохранение через Navbar

- **WHEN** пользователь нажимает активную «Сохранить»
- **THEN** вызывается POST /api/git/commit, при успехе показывается toast снизу с commit message, после обновления статуса кнопка становится неактивной или скрывается с активной зоны (нет незакоммиченных изменений)

### Requirement: Просмотр узла

UI ДОЛЖЕН (SHALL) позволять просматривать содержимое узла. Страница узла MUST содержать: кнопку «Назад»; breadcrumbs по path; панель действий (Action Bar) после breadcrumbs; заголовок из metadata.title; header с type badge, created, updated, keywords; строку атрибуции источника; блок аннотации и блок содержания с рендерингом markdown.

Панель действий (Action Bar) MUST объединять все управляющие действия: toggle «Проверено», кнопку «Перевести» (для article без перевода), разделитель, «Переместить», «Удалить». Кнопка «Удалить» MUST открывать модалку подтверждения (AlertDialog). Кнопка «Переместить» MUST открывать модалку перемещения с текстовым полем полного пути и деревом тем.

#### Сценарий: Открытие узла

- **WHEN** пользователь выбирает узел из таблицы
- **THEN** отображаются breadcrumbs, панель действий, заголовок, header с метаданными, аннотация и содержание

#### Сценарий: Панель действий для article

- **WHEN** открыт узел типа article без переводов
- **THEN** панель содержит: toggle «Проверено», «Перевести», разделитель, «Переместить», «Удалить»

#### Сценарий: Модалка подтверждения удаления

- **WHEN** пользователь нажимает «Удалить»
- **THEN** открывается AlertDialog с заголовком узла, путём, кнопками «Отмена» и «Удалить» (destructive)

#### Сценарий: Успешное удаление

- **WHEN** пользователь подтверждает удаление и API возвращает успех
- **THEN** переход на страницу обзора

#### Сценарий: Модалка перемещения с полным путём

- **WHEN** пользователь нажимает «Переместить»
- **THEN** открывается Dialog с текстовым полем «Новый путь» (предзаполнено текущим путём, редактируемое), деревом тем и кнопкой «Переместить»

#### Сценарий: Перемещение с созданием новых тем

- **WHEN** пользователь вводит путь с несуществующими темами и нажимает «Переместить»
- **THEN** API создаёт промежуточные директории, при успехе — переход на новую страницу узла

#### Сценарий: Выбор темы из дерева

- **WHEN** пользователь кликает тему в дереве
- **THEN** текстовое поле «Новый путь» обновляется: `{выбранная_тема}/{текущий_slug}`
