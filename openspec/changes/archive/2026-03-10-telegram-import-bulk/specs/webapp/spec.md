## MODIFIED Requirements

### Requirement: Раздел «Добавить»

При выборе «Добавить» MUST отображаться страница с табами: «Вручную» и «Импорт из Telegram».

**Вкладка «Вручную»:** форма с переключателем типа контента (авто, статья, ссылка, заметка), большое текстовое поле (textarea) и кнопка «Добавить». При выборе «статья» или «ссылка» MUST отображаться подсказка «Вставьте URL в текст». Форма предназначена для ввода текста нового знания и отправки в ingestion pipeline. При выборе типа, отличного от «авто», значение MUST передаваться в API как type_hint.

Во время обработки (loading) форма MUST блокировать повторную отправку: textarea и кнопка disabled, MUST отображаться анимация загрузки (спиннер или индикатор прогресса). Кнопка MUST показывать состояние обработки (напр. «Обработка...» или спиннер).

При успехе MUST отображаться блок (Alert/Banner) с сообщением «Добавлено» и ссылкой «Перейти к узлу» (если API вернул path). При ошибке MUST отображаться блок с текстом ошибки. Сообщение сбрасывается при следующей отправке.

**Вкладка «Импорт из Telegram»:** MUST отображаться возможность загрузить JSON-файл экспорта чата. После загрузки MUST отображаться индикатор прогресса (N/M, принято X, отклонено Y), карточка текущей записи (текст, source_author), переключатель типа контента, кнопки «Отклонить» и «Принять». При «Принять» MUST вызываться API accept с выбранным type_hint; при «Отклонить» — API reject. При наличии активной сессии (session_id в state или из предыдущей загрузки) MUST автоматически загружаться состояние сессии (GET session) и отображаться текущая запись. Должна быть возможность «Начать заново» для очистки сессии и загрузки нового файла.

#### Сценарий: Добавление знания

- **WHEN** пользователь на вкладке «Вручную» вводит текст и нажимает «Добавить»
- **THEN** текст отправляется в API (ingestion), отображается результат

#### Сценарий: Добавление с выбором типа

- **WHEN** пользователь на вкладке «Вручную» выбирает тип «статья», вводит текст и нажимает «Добавить»
- **THEN** текст и type_hint=article отправляются в API

#### Сценарий: Пустой ввод

- **WHEN** пользователь нажимает «Добавить» без ввода текста
- **THEN** кнопка неактивна или показывается валидация

#### Сценарий: Блокировка при обработке

- **WHEN** запрос на добавление выполняется (loading)
- **THEN** textarea и кнопка disabled, отображается анимация загрузки, повторная отправка невозможна

#### Сценарий: Подсказка при выборе статьи или ссылки

- **WHEN** пользователь выбирает тип «статья» или «ссылка»
- **THEN** под переключателем отображается текст «Вставьте URL в текст»

#### Сценарий: Успешное добавление

- **WHEN** запрос завершился успешно
- **THEN** отображается блок с сообщением «Добавлено» и ссылкой «Перейти к узлу» (если API вернул path)

#### Сценарий: Ошибка при добавлении

- **WHEN** запрос завершился с ошибкой
- **THEN** отображается блок с текстом ошибки из API

#### Сценарий: Загрузка JSON для импорта

- **WHEN** пользователь на вкладке «Импорт из Telegram» выбирает JSON-файл и загружает
- **THEN** создаётся сессия, отображается первая запись и прогресс

#### Сценарий: Принять запись при импорте

- **WHEN** пользователь нажимает «Принять» для текущей записи импорта
- **THEN** запись передаётся в ingestion, отображается следующая запись

#### Сценарий: Отклонить запись при импорте

- **WHEN** пользователь нажимает «Отклонить»
- **THEN** запись пропускается, отображается следующая

#### Сценарий: Восстановление сессии импорта

- **WHEN** пользователь открывает вкладку «Импорт из Telegram» и есть активная сессия (session_id)
- **THEN** загружается состояние сессии, отображается текущая запись и прогресс
