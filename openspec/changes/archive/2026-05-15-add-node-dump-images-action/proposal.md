## Why

На странице узла уже есть действие «Нормализация» с наблюдаемым процессом и логами, но для `dump images` такого UX нет. Пользователю нужен такой же управляемый запуск из UI и такая же прозрачность выполнения, чтобы извлечение изображений было предсказуемым и диагностируемым.

## What Changes

- Добавить в Action Bar страницы узла действие «Выгрузить изображения» для article-узлов.
- Добавить backend endpoint запуска операции `dump images` для конкретного узла и контракт асинхронного статуса.
- Добавить сбор и выдачу логов операции `dump images` по модели, эквивалентной логам нормализации (stdout/stderr/system + offset).
- Добавить в UI нижнюю панель логов операции `dump images` с инкрементальным обновлением.
- После успешного завершения `dump images` автоматически запускать `sync` по тому же принципу, что у нормализации.

## Capabilities

### New Capabilities
- `node-dump-images`: Серверный запуск `dump images` для одного узла с логированием и post-step sync.

### Modified Capabilities
- `node-action-panel`: Добавление кнопки «Выгрузить изображения» и панели логов операции на странице узла.
- `rest-api`: Добавление endpoint(ов) запуска/статуса/логов операции `dump images`.

## Impact

- `web/`: Action Bar страницы узла, состояния запуска/блокировки, нижняя панель логов `dump images`, API client.
- `internal/api`, `internal/bootstrap` и слой orchestration операций: запуск операции, хранение статуса, буфер логов, post-step sync.
- Тесты: API-тесты для endpoint-ов `dump images` и frontend-тесты для нового действия и лог-панели.
