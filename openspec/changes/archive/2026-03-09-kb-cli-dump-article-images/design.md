## Context

kb-cli уже поддерживает `validate` и `init`. База знаний хранит узлы как `{themePath}/{slug}.md`; вложения — в `{themePath}/{slug}/` (в т.ч. `images/`). Статьи часто содержат markdown-ссылки на удалённые изображения (`![alt](url)`), которые при оффлайн-доступе или удалении источника становятся недоступны.

## Goals / Non-Goals

**Goals:**
- Добавить подкоманду `dump-images` в kb-cli
- Скачивать удалённые изображения из markdown и сохранять локально
- Заменять URL на относительные пути к локальным файлам
- Обеспечить идемпотентность (повторный вызов не дублирует файлы)

**Non-Goals:**
- Обработка встроенных base64-изображений
- Обработка ссылок в HTML-блоках (только markdown-синтаксис)
- Автоматический запуск при ingestion (отдельная задача)

## Decisions

### 1. Парсинг markdown-ссылок на изображения

**Решение:** Regex для `![alt](url)` и опционально `[text](url)` с проверкой расширения (.png, .jpg, .jpeg, .gif, .webp, .svg).

**Альтернатива:** Полноценный markdown-парсер (goldmark и т.п.) — избыточен для задачи извлечения URL.

### 2. Именование скачанных файлов

**Решение:** Использовать хеш URL (SHA256, первые 12 символов) + оригинальное расширение. Это даёт: уникальность, дедупликацию, читаемость.

**Альтернатива:** Оригинальное имя из URL — часто длинное, с query-параметрами, небезопасное.

### 3. Путь сохранения

**Решение:** `{themePath}/{slug}/images/{hash}.{ext}`. Директория `{slug}/` создаётся при необходимости. Соответствует knowledge-storage: вложения в `{themePath}/{slug}/`.

### 4. Относительный путь в markdown

**Решение:** Для плоской структуры (`slug.md` рядом с `slug/`) — путь `{slug}/images/{filename}`. Markdown и images в одной теме, путь от корня темы.

### 5. Флаги CLI

- `--path` / `-p`: путь к базе (как в validate/init)
- `--file` / `-f`: путь к конкретному .md (если указан — path опционально, определяется по родительской директории)
- `--dry-run`: только вывести список URL и целевые пути, не скачивать и не менять файл

### 6. Обработка ошибок

- Ошибка загрузки одного изображения — логировать, продолжать остальные
- В конце — ненулевой exit code, если хотя бы одна загрузка не удалась

## Risks / Trade-offs

| Риск | Митигация |
|------|-----------|
| URL ведёт на редирект или HTML | Проверять Content-Type ответа; отклонять не-image |
| Большие файлы | Ограничение размера (например, 10 MB) — конфигурируемо |
| Таймаут сети | Использовать http.Client с разумным timeout (30s) |
| Невалидный markdown | Работать с сырым текстом; regex не ломает структуру |

## Migration Plan

Новая функциональность, миграция не требуется. Пользователи вызывают `kb-cli dump-images` по необходимости.

### 7. Обновление frontmatter

**Решение:** При изменении markdown обновлять поле `updated` во frontmatter (ISO 8601). Это сохраняет согласованность с git history и валидацией базы.
