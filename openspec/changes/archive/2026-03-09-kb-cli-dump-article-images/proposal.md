## Why

Статьи в базе знаний часто содержат ссылки на удалённые изображения (например, с Habr, Medium, CDN). При оффлайн-доступе или при удалении исходных ресурсов картинки становятся недоступны. Нужна возможность «зафиксировать» статью: скачать все внешние изображения локально и заменить ссылки на локальные пути — чтобы база оставалась самодостаточной и оффлайн-first.

## What Changes

- Добавить в kb-cli подкоманду `dump-images` (или `dump-article-images`)
- Принимать путь к .md файлу статьи (или путь к базе + slug)
- Находить в markdown все ссылки на удалённые изображения (формат `![alt](url)` и `[text](url)` с расширениями изображений)
- Скачивать изображения по HTTP(S)
- Сохранять в директорию вложений узла (`{themePath}/{slug}/images/`)
- Заменять URL в markdown на относительные локальные пути
- Обрабатывать дубликаты (одинаковые URL — один файл)
- Поддерживать флаги: `--path` (база), `--file` (конкретный .md), `--dry-run` (только показать, что будет сделано)

## Capabilities

### New Capabilities

- `kb-cli-dump-images`: подкоманда kb-cli для дампа удалённых изображений из статьи в локальную директорию узла и замены ссылок в markdown

### Modified Capabilities

- `kb-cli`: добавить требование на подкоманду dump-images (delta spec)

## Impact

- **cmd/kb-cli**: новая подкоманда, новый файл `dump_images.go` (или аналог)
- **internal/kb** (опционально): утилиты для парсинга markdown и извлечения URL изображений
- Зависимости: возможно `net/http` для загрузки, парсинг markdown (regex или библиотека)
- База данных: изменения в .md файлах и создание файлов в `{slug}/images/`
