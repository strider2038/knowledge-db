## Purpose

Подкоманда kb-cli для дампа удалённых изображений из статьи: скачивание по URL, сохранение в директорию узла и замена ссылок в markdown на локальные пути. Обеспечивает оффлайн-доступ к изображениям статей.

## Requirements

### Requirement: Подкоманда dump-images

kb-cli ДОЛЖЕН (SHALL) предоставлять подкоманду dump-images для скачивания удалённых изображений из markdown-статьи и замены ссылок на локальные пути.

#### Сценарий: Dump по пути к файлу

- **WHEN** kb-cli dump-images --file /path/to/base/theme/article.md
- **THEN** система находит все URL изображений в markdown, скачивает их в `{themePath}/{slug}/images/`, заменяет ссылки на относительные пути и сохраняет обновлённый markdown

#### Сценарий: Dump с указанием базы

- **WHEN** kb-cli dump-images --path /path/to/base --file theme/article.md (или slug)
- **THEN** система обрабатывает указанный файл относительно пути базы

#### Сценарий: Dry-run

- **WHEN** kb-cli dump-images --file article.md --dry-run
- **THEN** система выводит список найденных URL и целевых путей без скачивания и изменения файла

### Requirement: Извлечение URL изображений

Система ДОЛЖНА (SHALL) находить в markdown ссылки формата `![alt](url)` и опционально `[text](url)` с расширениями изображений (.png, .jpg, .jpeg, .gif, .webp, .svg). Локальные пути (относительные, без scheme) MUST игнорироваться.

#### Сценарий: Markdown-изображение

- **WHEN** в теле есть `![Diagram](https://example.com/diagram.png)`
- **THEN** URL извлекается для загрузки

#### Сценарий: Локальная ссылка игнорируется

- **WHEN** в теле есть `![Local](images/local.png)` или `![Local](./images/local.png)`
- **THEN** ссылка не обрабатывается (уже локальная)

### Requirement: Сохранение и именование файлов

Скачанные изображения MUST сохраняться в `{themePath}/{slug}/images/`. Имя файла — хеш URL (первые 12 символов SHA256) + оригинальное расширение. Одинаковые URL MUST сохраняться в один файл (дедупликация).

#### Сценарий: Уникальное имя

- **WHEN** скачивается https://example.com/image.png
- **THEN** файл сохраняется как `images/{hash}.png`

#### Сценарий: Дубликат URL

- **WHEN** один и тот же URL встречается в статье дважды
- **THEN** скачивается один раз, обе ссылки заменяются на один локальный путь

### Requirement: Замена ссылок в markdown

После успешной загрузки система MUST заменить каждый URL в markdown на относительный путь к локальному файлу в формате `{slug}/images/{filename}`.

#### Сценарий: Замена после загрузки

- **WHEN** изображение скачано в `theme/slug/images/abc123.png`
- **THEN** в markdown `![alt](https://...)` заменяется на `![alt](slug/images/abc123.png)`

### Requirement: Обновление frontmatter

При изменении markdown-файла система MUST обновлять поле `updated` во frontmatter (ISO 8601).

#### Сценарий: Обновление даты при замене ссылок

- **WHEN** хотя бы одна ссылка успешно заменена на локальный путь
- **THEN** поле `updated` во frontmatter обновлено на текущее время (ISO 8601)

### Requirement: Обработка ошибок загрузки

При ошибке загрузки одного изображения система MUST логировать ошибку и продолжать обработку остальных. При наличии хотя бы одной неудачной загрузки exit code MUST быть ненулевым.

#### Сценарий: Частичный успех

- **WHEN** из 3 изображений одно не удалось скачать (404, timeout)
- **THEN** 2 успешно загружены и заменены, 1 ссылка остаётся без изменений, exit code != 0
