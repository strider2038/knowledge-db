## Purpose

Telegram-бот для приёма сообщений и URL, вызова ingestion pipeline. Работает в том же процессе, что и kb-server. Принимает сообщения только от авторизованного пользователя.

## Requirements

## ADDED Requirements

### Requirement: Метаданные источника при пересылке

При пересылке сообщения бот ДОЛЖЕН (SHALL) извлекать из `forward_origin` автора и ссылку на оригинал (если доступна) и передавать их в ingestion pipeline вместе с контентом. Система MUST парсить типы MessageOrigin (channel, user, chat, hidden_user) и извлекать: для канала — ссылку https://t.me/... и автора (название канала, @username или author_signature); для user/chat/hidden_user — автора (username, имя, название чата).

#### Сценарий: Пересланный пост из канала

- **WHEN** пользователь пересылает боту сообщение из канала (forward_origin.type = "channel")
- **THEN** бот извлекает ссылку на пост и автора (название канала или @username), передаёт их в IngestText вместе с контентом

#### Сценарий: Пересланное сообщение от пользователя

- **WHEN** пользователь пересылает боту сообщение от другого пользователя (forward_origin.type = "user" или "hidden_user")
- **THEN** бот извлекает автора (username или sender_user_name), передаёт в IngestText; ссылка может быть пустой (Telegram не даёт публичной ссылки на сообщения из лички)

#### Сценарий: Reply на пересланное с метаданными

- **WHEN** пользователь отправляет reply с комментарием на пересланное сообщение
- **THEN** бот извлекает метаданные источника из reply_to_message.forward_origin и передаёт их в IngestText вместе с объединённым текстом

## MODIFIED Requirements

### Requirement: Приём сообщений

Бот MUST принимать текстовые сообщения только от авторизованного пользователя и передавать их в ingestion pipeline через IngestText (с опциональными метаданными источника при пересылке). Бот не выполняет предварительный анализ контента — LLM-оркестратор внутри pipeline определяет тип контента (текст, URL на статью, URL на ресурс) и выбирает действия. При пересылке сообщения с комментарием (reply) бот MUST объединять пару в единый input: комментарий + контент пересланного сообщения (текст, caption, URL и т.п.). При пересылке бот MUST передавать метаданные источника (ссылка, автор) в ingestion.

#### Сценарий: Получение текста

- **WHEN** пользователь отправляет боту текст
- **THEN** бот передаёт текст в IngestText и отправляет подтверждение с результатом

#### Сценарий: Получение URL

- **WHEN** пользователь отправляет боту сообщение с URL
- **THEN** бот передаёт всё сообщение в IngestText (LLM внутри pipeline определит, что делать с URL)

#### Сценарий: Получение текста с инструкциями

- **WHEN** пользователь отправляет "https://habr.com/... сохрани в go/concurrency"
- **THEN** бот передаёт всё сообщение в IngestText (LLM учтёт инструкции)

#### Сценарий: Пересланное медиа с подписью

- **WHEN** пользователь пересылает боту сообщение с медиа (фото, видео, документ) и подписью
- **THEN** бот передаёт текст подписи (caption) в IngestText с метаданными источника и отправляет подтверждение

#### Сценарий: Пересланное сообщение с комментарием (reply)

- **WHEN** пользователь пересылает боту сообщение и затем отправляет reply с комментарием (или наоборот — reply с reply_to_message на пересланное)
- **THEN** бот объединяет с явными метками («Инструкции пользователя: …», «Пересланное сообщение: …»), передаёт в IngestText с метаданными источника, логирует объединение на уровне info и отправляет одно подтверждение с результатом
