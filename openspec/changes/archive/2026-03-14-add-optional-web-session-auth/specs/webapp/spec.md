## Purpose

Определить поведение web UI при включённой опциональной сессионной авторизации.

## Requirements

## ADDED Requirements

### Requirement: Экран входа и проверка сессии

Web UI ДОЛЖЕН (SHALL) поддерживать страницу входа и проверку статуса сессии через `GET /api/auth/session` при запуске в режиме включённой авторизации. При отсутствии валидной сессии UI MUST направлять пользователя на страницу входа.

#### Scenario: Пользователь без сессии открывает защищённый раздел

- **WHEN** пользователь открывает раздел, требующий авторизацию, и `GET /api/auth/session` возвращает неаутентифицированный статус
- **THEN** UI перенаправляет пользователя на страницу входа

#### Scenario: Пользователь с активной сессией открывает приложение

- **WHEN** пользователь открывает приложение и `GET /api/auth/session` подтверждает валидную сессию
- **THEN** UI отображает запрошенный раздел без редиректа на login

### Requirement: Поведение login/logout flow

Web UI MUST отправлять credentials на `POST /api/auth/login`, сохранять пользовательский контекст перехода и после успешного входа возвращать пользователя на исходный маршрут. UI MUST поддерживать выход через `POST /api/auth/logout` с последующим переходом на login.

#### Scenario: Успешный вход

- **WHEN** пользователь вводит корректные login/password и отправляет форму входа
- **THEN** UI вызывает `POST /api/auth/login` и перенаправляет пользователя на исходный или дефолтный маршрут

#### Scenario: Ошибка входа

- **WHEN** пользователь отправляет неверные credentials
- **THEN** UI отображает сообщение об ошибке и остаётся на странице login

#### Scenario: Выход из сессии

- **WHEN** пользователь инициирует logout
- **THEN** UI вызывает `POST /api/auth/logout` и переводит пользователя на страницу входа
