## Why

Текущее разделение на два бинарника (`kb-server` и `kb-cli`) усложняет эксплуатацию в Docker и интеграцию с Cursor Agent: для одной рабочей среды нужно собирать, доставлять и вызывать два разных исполняемых файла. Объединение в один бинарник уменьшает операционную сложность и делает интерфейс запуска единообразным.

## What Changes

- **BREAKING**: заменить два входных бинарника (`kb-server`, `kb-cli`) на единый бинарник `kb`.
- **BREAKING**: заменить CLI-фреймворк `cobra` на `urfave/cli`.
- Добавить подкоманду `serve` в `kb` для запуска серверной части (HTTP API, UI, фоновые процессы).
- Перенести существующие CLI-подкоманды (`validate`, `init`, `dump-images`, `expand-urls`, `reindex-links`) в единый root command `kb`.
- Обновить сборку, Docker-конфигурации и документацию на использование нового интерфейса команд.
- Не сохранять обратную совместимость со старыми именами бинарников и командами запуска.

## Capabilities

### New Capabilities
- `kb-unified-binary`: Единая точка входа `kb` с серверной подкомандой `serve` и операционными CLI-подкомандами.

### Modified Capabilities
- `kb-cli`: Требования к CLI обновляются с учётом нового имени бинарника и работы как части единого `kb`.

## Impact

- Backend entrypoints в `cmd/` (структура и инициализация команд).
- CLI-зависимости в `go.mod` (`cobra` -> `urfave/cli`) и связанный код инициализации команд.
- Скрипты сборки и деплоя (включая Dockerfile/compose/CI, где упоминаются старые бинарники).
- Пользовательская и эксплуатационная документация (`README.md`, примеры запуска).
- Тесты CLI/интеграционные сценарии запуска сервера через командный интерфейс.
