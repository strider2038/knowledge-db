## ADDED Requirements

### Requirement: Единый исполняемый файл kb
Система ДОЛЖНА (SHALL) предоставлять единый исполняемый файл `kb` как единственную поддерживаемую точку входа для серверного и операционного режимов.

#### Scenario: Доступность общего help
- **WHEN** пользователь выполняет `kb --help`
- **THEN** отображается корневая справка с перечнем подкоманд, включая `serve` и технические команды

### Requirement: Подкоманда serve для запуска серверной части
Командный интерфейс MUST предоставлять подкоманду `serve`, которая запускает серверную часть приложения с теми же runtime-возможностями, что и ранее отдельный `kb-server`.

#### Scenario: Запуск сервера через единый бинарник
- **WHEN** пользователь выполняет `kb serve`
- **THEN** запускается HTTP API, embedded UI и зарегистрированные фоновые процессы в соответствии с конфигурацией окружения

### Requirement: Отсутствие поддержки legacy-бинарников
После миграции система MUST не требовать и не документировать использование `kb-server` и `kb-cli` как поддерживаемых entrypoint-ов.

#### Scenario: Операционная документация
- **WHEN** пользователь открывает актуальные инструкции запуска и деплоя
- **THEN** все примеры используют только команды единого бинарника `kb`

### Requirement: CLI на базе urfave/cli
Командный интерфейс `kb` ДОЛЖЕН (SHALL) быть реализован на библиотеке `urfave/cli`, а зависимость от `cobra` MUST быть удалена из поддерживаемого runtime-пути команд.

#### Scenario: Проверка технологической основы CLI
- **WHEN** выполняется сборка проекта после миграции
- **THEN** команды `kb` и `kb serve` работают через `urfave/cli`, а основной код entrypoint не использует `cobra`
