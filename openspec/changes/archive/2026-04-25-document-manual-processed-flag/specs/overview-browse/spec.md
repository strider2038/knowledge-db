## Purpose

Дельта к обзору: фильтр по признаку ручной проверки и параметр в URL.

## Requirements

## MODIFIED Requirements

### Requirement: Состояние в URL

Фильтры (path, type, q, sort, order, page, manual_processed) MUST сохраняться в query-параметрах URL. При загрузке страницы с URL состояние восстанавливается. Параметр manual_processed SHALL иметь значения: отсутствует (все узлы), `true` (только отмеченные вручную), `false` (только неотмеченные).

#### Сценарий: Восстановление

- **WHEN** пользователь открывает страницу с ?path=ai&type=article&q=go
- **THEN** отображается поддерево ai, фильтр article, поиск «go»

#### Сценарий: Восстановление с фильтром проверки

- **WHEN** пользователь открывает страницу с ?path=&recursive=true&manual_processed=false
- **THEN** таблица показывает только узлы без отметки ручной проверки

## ADDED Requirements

### Requirement: Фильтр по ручной проверке

UI ДОЛЖЕН (SHALL) предоставлять в области фильтров обзора (рядом с поиском и фильтрами по типу) выбор состояния ручной проверки: «Все», «Проверено вручную», «Не проверено». Выбор MUST маппиться на query-параметр manual_processed (отсутствует / true / false) при запросе GET /api/nodes.

#### Сценарий: Только непроверенные

- **WHEN** пользователь выбирает «Не проверено»
- **THEN** в запрос списка узлов передаётся manual_processed=false, таблица обновляется

#### Сценарий: Только проверенные

- **WHEN** пользователь выбирает «Проверено вручную»
- **THEN** в запрос передаётся manual_processed=true

#### Сценарий: Все записи

- **WHEN** пользователь выбирает «Все»
- **THEN** параметр manual_processed не передаётся в API
