## Purpose

Дельта: фильтрация и отображение личных меток на странице «Обзор», компактная панель фильтров.

## MODIFIED Requirements

### Requirement: Состояние в URL

Фильтры (path, type, q, sort, order, page, manual_processed, labels) MUST сохраняться в query-параметрах URL. При загрузке страницы с URL состояние восстанавливается. Параметр manual_processed SHALL иметь значения: отсутствует (все узлы), `true` (только отмеченные вручную), `false` (только неотмеченные). Параметр labels SHALL быть списком меток через запятую (пустой — фильтр выключен); семантика фильтра на API — AND (все перечисленные метки на узле).

#### Сценарий: Восстановление

- **WHEN** пользователь открывает страницу с ?path=ai&type=article&q=go
- **THEN** отображается поддерево ai, фильтр article, поиск «go»

#### Сценарий: Восстановление с фильтром проверки

- **WHEN** пользователь открывает страницу с ?path=&recursive=true&manual_processed=false
- **THEN** таблица показывает только узлы без отметки ручной проверки

#### Сценарий: Восстановление с фильтром меток

- **WHEN** пользователь открывает страницу с ?labels=favorite,review
- **THEN** таблица и дерево применяют фильтр узлов с обеими метками

### Requirement: Фильтр по ручной проверке

UI ДОЛЖЕН (SHALL) предоставлять фильтр ручной проверки в общей панели фильтров обзора. На десктопе (md+) и в мобильном Sheet MUST использоваться кнопка-триггер dropdown: подпись отражает текущее состояние («Проверка» — все, «Проверено», «Не проверено»); меню содержит пункты «Все записи», «Проверено вручную», «Не проверено». Выбор MUST маппиться на query-параметр manual_processed (отсутствует / true / false) при запросе GET /api/nodes.

#### Сценарий: Только непроверенные

- **WHEN** пользователь в dropdown выбирает «Не проверено»
- **THEN** в запрос списка узлов передаётся manual_processed=false, таблица обновляется, на кнопке отображается «Не проверено»

#### Сценарий: Только проверенные

- **WHEN** пользователь выбирает «Проверено вручную»
- **THEN** в запрос передаётся manual_processed=true, на кнопке — «Проверено»

#### Сценарий: Все записи

- **WHEN** пользователь выбирает «Все записи»
- **THEN** параметр manual_processed не передаётся в API, на кнопке — «Проверка»

### Requirement: Таблица

Таблица MUST содержать колонки: Название (из title), Тип (цветом), Дата, Метки (цветные чипы личных labels), Ссылка (для link/article — кликабельная, target="_blank"). Сортировка по клику на заголовок. Кнопки фильтров по типам MUST использовать те же цвета, что и badge типа в таблице (article=синий, link=зелёный, note=серый).

#### Сценарий: Сортировка

- **WHEN** пользователь кликает на заголовок колонки
- **THEN** данные сортируются по этой колонке; направление (asc/desc) переключается

#### Сценарий: Ссылка

- **WHEN** узел типа link или article имеет source_url
- **THEN** в колонке отображается кликабельная ссылка, открывающаяся в новой вкладке

## ADDED Requirements

### Requirement: Компактная панель фильтров (десктоп)

На viewport md и шире UI MUST показывать под строкой поиска одну горизонтальную панель фильтров (toolbar): toggle типов, dropdown ручной проверки, чипы выбранных меток и кнопка «+ Метка», при необходимости — «Сбросить». Панель MAY переноситься на несколько строк (flex-wrap). Отдельная бордерная секция с вертикальными заголовками блоков фильтров MUST NOT использоватьаться на десктопе.

#### Сценарий: Фильтры всегда под поиском

- **WHEN** пользователь открывает «Обзор» на широком экране
- **THEN** типы, проверка и метки доступны сразу под полем поиска без открытия Sheet

### Requirement: Фильтры на мобильных устройствах

Ниже breakpoint md вторичные фильтры (тип, ручная проверка, метки) MUST быть скрыты из основного вида. Рядом с поиском MUST быть кнопка «Фильтры» (с badge числа активных фильтров), открывающая Sheet с той же панелью фильтров. При активных фильтрах и закрытом Sheet MUST отображаться строка «Активно:» с чипами для снятия каждого фильтра и «Сбросить всё».

#### Сценарий: Настройка в Sheet

- **WHEN** пользователь на узком экране нажимает «Фильтры»
- **THEN** открывается Sheet со всеми фильтрами и кнопкой сброса внизу

#### Сценарий: Активные фильтры без Sheet

- **WHEN** на мобильном выбраны метки и Sheet закрыт
- **THEN** под поиском видны чипы активных фильтров для быстрого снятия

### Requirement: Фильтр по личным меткам

UI ДОЛЖЕН (SHALL) предоставлять выбор одной или нескольких меток (личных labels) в панели фильтров. Выбранные метки MUST отображаться цветными чипами с возможностью снятия. Добавление — кнопка «+ Метка» и dropdown: поле поиска/ввода, список подсказок из GET /api/label-suggestions (чекбоксы, цветные чипы в списке), опционально пункт «Добавить …» для новой метки. Выбранные метки MUST передаваться в GET /api/nodes как `labels` (через запятую). Фильтрация — AND на сервере. При активном фильтре по меткам дерево тем MUST сужаться по всей базе (path не ограничивает набор путей для дерева), по тому же принципу, что фильтр по типу и manual_processed.

#### Сценарий: Выбор одной метки

- **WHEN** пользователь включает метку «favorite» в dropdown
- **THEN** в запрос списка узлов передаётся labels=favorite, появляется цветной чип, таблица и дерево обновляются

#### Сценарий: Несколько меток (AND)

- **WHEN** пользователь выбирает метки «favorite» и «review»
- **THEN** в запрос передаётся labels=favorite,review и отображаются только узлы с обеими метками

#### Сценарий: Сброс фильтра меток

- **WHEN** пользователь снимает все выбранные метки
- **THEN** параметр labels не передаётся в API, дерево возвращается к полному виду (с учётом других активных фильтров)

### Requirement: Отображение меток в таблице обзора

Таблица обзора MUST отображать метки узла в колонке «Метки» (чипы). Цвет каждого чипа MUST определяться стабильно по строке метки (hash → палитра), без отдельного конфиг-файла в базе. Метки MUST визуально отличаться от badge типа контента (article/link/note).

#### Сценарий: Узел с метками

- **WHEN** узел имеет labels `["favorite", "hotspot"]`
- **THEN** в таблице отображаются два цветных чипа с текстом меток

#### Сценарий: Узел без меток

- **WHEN** у узла labels пустой
- **THEN** колонка меток пуста без ошибки
