## Why

В заметках и статьях часто остаются короткие редиректные ссылки (bit.ly, clc.to и т.д.) и URL с UTM/рекламными query-параметрами. Это ухудшает читаемость, усложняет поиск дубликатов и не даёт стабильного «канонического» URL. Нужна оффлайн-совместимая утилита на стороне CLI, чтобы массово или точечно приводить ссылки в markdown к полному адресу и убирать мусорные параметры — без обязательного запуска сервера.

## What Changes

- Добавить в kb-cli подкоманду (например `expand-urls`) для обработки одного или нескольких `.md` файлов внутри базы
- Для каждого найденного `http`/`https` URL вызывать ту же логику нормализации, что и при импорте: следование редиректам (раскрытие сокращений) и удаление UTM-параметров
- Заменять URL в теле markdown (как минимум в `[text](url)` и совпадениях, покрываемых общей логикой)
- Флаги: `--path` (корень базы), `--file` (конкретный файл), опционально обход дерева; `--dry-run` для предпросмотра замен без записи

## Capabilities

### New Capabilities

- `kb-cli-expand-urls`: подкоманда kb-cli для раскрытия редиректных ссылок и очистки query от UTM/трекинговых параметров в markdown

### Modified Capabilities

- `kb-cli`: добавить требование на новую подкоманду expand-urls (delta spec)

## Impact

- **cmd/kb-cli**: новая подкоманда, регистрация в `main.go`
- **internal/pkg/urlutil**: переиспользование `NormalizeURL` (и при необходимости расширение снятия трекинг-параметров)
- **internal/kb** (или отдельный пакет): извлечение и замена URL в markdown, тесты
- Сеть: HEAD/редиректы с таймаутом; при ошибках — сохранение исходного URL (как в `NormalizeURL`)
