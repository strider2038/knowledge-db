## Why

При пересылке сообщения с комментарием в Telegram пользователь отправляет два отдельных update: само пересланное сообщение и отдельный комментарий (reply). Бот получает их как независимые события и обрабатывает по отдельности — из-за чего теряется контекст. Комментарий (инструкция, теги, путь) и пересланный контент должны обрабатываться вместе как единый input в ingestion pipeline.

## What Changes

- Логика объединения связанных update: если комментарий пользователя является reply на пересланное сообщение (или наоборот), обрабатывать пару как один контекст
- Передача в IngestText объединённого текста: комментарий + контент пересланного сообщения (текст, caption, URL и т.п.)
- Один цикл подтверждения и итогового ответа на пару сообщений вместо двух независимых

## Impact

- **cmd/kb-server**: обработчик Telegram-бота — логика связывания update, буферизация/ожидание reply в пределах окна времени
- **internal/ingestion**: без изменений — IngestText принимает текст, формат объединённого контента определяется на стороне бота
