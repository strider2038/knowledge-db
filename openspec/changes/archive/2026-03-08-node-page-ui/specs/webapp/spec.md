## Purpose

Веб-интерфейс для просмотра, поиска, создания и редактирования записей базы знаний. React + Vite, статика встраивается в бинарник kb-server.

## Requirements

## MODIFIED Requirements

### Requirement: Раздел «Обзор»

При выборе «Обзор» MUST отображаться страница обзора базы: слева — иерархическое дерево тем (фильтр по поддереву), справа — таблица узлов с колонками Название, Тип, Дата, Ссылка. Дерево поддерживает «Вся база»; таблица — фильтры по типам (article/link/note), поиск по тексту, сортировку, пагинацию. Детали — в спецификации overview-browse. При переходе на страницу узла ссылка MUST передавать state с returnTo (текущий pathname + search) для сохранения контекста обзора.

#### Сценарий: Навигация по дереву

- **WHEN** пользователь раскрывает/выбирает тему в дереве
- **THEN** таблица справа обновляется и показывает узлы выбранного поддерева (рекурсивно)

#### Сценарий: Отображение списка узлов

- **WHEN** выбрана тема, подтема или «Вся база»
- **THEN** в таблице отображаются узлы с title, type, path, датой, ссылкой (для link/article)

#### Сценарий: Переход на страницу узла с сохранением контекста

- **WHEN** пользователь кликает по узлу в таблице
- **THEN** переход на страницу узла с state.returnTo, содержащим URL обзора с текущими фильтрами (path, type, q, sort, order, page)

### Requirement: Просмотр узла

UI ДОЛЖЕН (SHALL) позволять просматривать содержимое узла. Страница узла MUST содержать: кнопку «Назад» (возврат на обзор с сохранением фильтров при переходе из обзора, иначе на главную); breadcrumbs по path (сегменты — ссылки на обзор с path-фильтром); заголовок из metadata.title с fallback на slug из path; header с type badge, created, updated, keywords (чипсы); при наличии source_url, source_author или source_date — отдельную строку атрибуции источника (ссылка на source_url, автор, дата источника) для всех типов узлов (article, link, note); отдельный блок аннотации и блок содержания с рендерингом markdown (таблицы, подсветка кода). Метаданные НЕ отображаются отдельным блоком. При наличии переводов MUST отображаться выбор языка (оригинал / переводы).

**Особенности типа link:** Узлы типа link (ссылка-закладка) НЕ имеют тела контента — блок «Содержание» MUST НЕ отображаться. Перед блоком аннотации MUST отображаться явная кликабельная ссылка на source_url (если задана): карточка/кнопка с иконкой и URL, target="_blank".

#### Сценарий: Открытие узла

- **WHEN** пользователь выбирает узел из таблицы
- **THEN** отображаются breadcrumbs, заголовок (title), header с метаданными, аннотация и содержание с рендерингом markdown

#### Сценарий: Выбор языка

- **WHEN** у узла есть переводы (translations)
- **THEN** отображается переключатель для выбора оригинала или перевода

#### Сценарий: Кнопка «Назад» при переходе из обзора

- **WHEN** пользователь перешёл на страницу узла из обзора и нажимает «Назад»
- **THEN** возврат на страницу обзора с теми же query-параметрами (path, type, q, sort, order, page)

#### Сценарий: Кнопка «Назад» при прямом заходе

- **WHEN** пользователь открыл страницу узла напрямую (закладка, ссылка) и нажимает «Назад»
- **THEN** переход на главную (/)

#### Сценарий: Breadcrumbs

- **WHEN** отображается страница узла с path «programming/scaling/load-balancing»
- **THEN** отображаются breadcrumbs: Обзор > programming > scaling > load-balancing, где каждый сегмент — ссылка на обзор с ?path=накопленный_путь

#### Сценарий: Markdown в аннотации и содержании

- **WHEN** аннотация или содержание содержат markdown (заголовки, списки, таблицы, код)
- **THEN** контент рендерится как markdown с поддержкой таблиц (GFM) и подсветки синтаксиса кода

#### Сценарий: Узел типа link

- **WHEN** пользователь открывает узел типа link с source_url
- **THEN** перед аннотацией отображается явная кликабельная ссылка на source_url; блок «Содержание» НЕ отображается

#### Сценарий: Атрибуция источника

- **WHEN** у узла заданы source_url, source_author и/или source_date (любой тип: article, link, note)
- **THEN** под строкой с type badge и датами отображается отдельная строка: кликабельная ссылка на source_url (если есть), автор (если есть), дата источника (если есть)
