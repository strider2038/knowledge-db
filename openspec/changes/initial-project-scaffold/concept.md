# Концепция базы знаний

## Хранение

База знаний хранится в директории под git.

### Структура

- Разбивка по темам (директории)
- Внутри тем — подтемы, всего 2–3 уровня вложенности
- Узлы — директории со статьями/заметками

### Структура узла (папка)

Каждый узел содержит:

- `annotation.md` — короткая аннотация статьи
- `content.md` — основное содержание статьи
- Дополнительные `.md` файлы — неограниченное число (в подпапке `notes/` для порядка)
- Вспомогательные файлы (sha-хеш статьи, смысловой вектор и т.п.) — в `.local/`, исключены из git
- Изображения — в подпапке `images/`
- `metadata.json` — ключевые слова, источник, даты и т.п.

### Пример структуры

```
data/
├── topic-a/
│   ├── subtopic-1/
│   │   └── node-name/
│   │       ├── annotation.md
│   │       ├── content.md
│   │       ├── metadata.json
│   │       ├── images/
│   │       ├── notes/
│   │       │   └── ...
│   │       └── .local/      # gitignore
│   │           ├── sha.txt
│   │           └── embedding.json
│   └── subtopic-2/
│       └── ...
└── topic-b/
    └── ...
```

## Консольная утилита (kb-cli)

Консольная утилита для локальной работы с базой:

1. **Валидация структуры** — проверка, что директория базы соответствует формату (темы, узлы, обязательные файлы annotation.md, content.md, metadata.json и т.п.)
2. **Установка agent skills** — копирование/настройка skill'ов под конкретную базу знаний (путь к data/, параметры)

Запуск: `kb validate <path>`, `kb install-skills <path>` (или аналогичные подкоманды).

## Как заполняется база

1. **Локально через агента** (Cursor, Claude и т.п.) — skill с инструкциями, установка через `kb install-skills`
2. **Пересылка сообщения в Telegram** — бот принимает и обрабатывает
3. **Вручную через веб-интерфейс**

## Процесс заполнения

### 1. Текст

Пользователь передаёт текст → агент/приложение с помощью LLM создаёт нужные файлы (annotation, content, metadata и т.д.).

### 2. Ссылка на статью (Habr и т.п.)

Пользователь передаёт URL → агент/приложение:

- забирает страницу
- конвертирует в markdown
- создаёт файлы узла
- добавляет ссылку на источник в метаданные

### 3. Видео (будущее)

Передача видео → транскрипция → генерация статьи/знания.
