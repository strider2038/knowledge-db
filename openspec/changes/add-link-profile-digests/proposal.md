## Why

Ссылки в базе знаний сейчас часто сохраняются как `type: link` с пустым телом и короткой аннотацией, поэтому RAG-чат получает только карточку ресурса, но не концептуальный контекст. Это особенно заметно для репозиториев, документации, длинных статей, сервисов и новостей, где полезно хранить не полный оригинал, а сжатое смысловое описание.

## What Changes

- Добавить модель профилей внешних источников: `source_kind` описывает природу источника, а `content_profile` описывает локальную форму знания.
- Научить ingestion отличать закладку на ресурс от концептуальной заметки по источнику: длинные статьи без полного копирования сохранять как `type=note` с `content_profile=conceptual_digest`, репозитории и сервисы — как `type=link` с профильным digest-телом.
- Генерировать markdown-тело для link/note digest из README, метаданных или извлечённого контента, отбрасывая технический шум вроде установки, quick start, команд и API-примеров.
- Добавить на странице узла действие «Обновить описание из источника», которое точечно применяет новый алгоритм к существующей ноде с `source_url`.
- Расширить индексирование, чтобы тело `type=link` с digest и концептуальные `type=note` были видимы для keyword/vector/chunk retrieval и попадали в RAG-контекст.
- Сохранить возможность хранить полную копию статьи как `type=article`, если пользователь явно хочет локальную копию.

## Capabilities

### New Capabilities

- `link-profile-digests`: правила классификации внешних источников и генерации концептуальных digest-тел для ссылок, репозиториев, документации, сервисов, длинных статей и новостей.

### Modified Capabilities

- `knowledge-storage`: добавить опциональные frontmatter-поля для профиля источника и локальной формы знания.
- `ingestion-pipeline`: изменить поведение URL ingestion для классификации источника, выбора `type` и генерации digest-тела.
- `embedding-index`: индексировать digest-тела link/note узлов так, чтобы они участвовали в RAG retrieval.
- `rest-api`: добавить endpoint для обновления описания существующего узла из `source_url`.
- `node-action-panel`: добавить действие обновления описания из источника на странице узла.

## Impact

- Backend ingestion: LLM tool schema, prompts, URL metadata/content fetch flow, создание узлов.
- REST API: новый endpoint для регенерации описания/digest существующего узла.
- Web UI: новая кнопка действия на странице узла и состояния loading/success/error.
- Storage/validation: поддержка новых опциональных frontmatter-полей без нарушения совместимости существующих узлов.
- Index/RAG: node embedding/searchable text/chunk indexing для link/note digest.
- CLI/API compatibility: существующие `type=article|link|note` остаются валидными; новые поля опциональны.
