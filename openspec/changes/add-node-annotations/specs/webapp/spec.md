## Purpose

Дельта: UI панели «Мои заметки» на странице узла.

## ADDED Requirements

### Requirement: Панель личных заметок на странице узла

UI SHALL предоставлять на странице узла панель «Мои заметки» для просмотра и редактирования личных аннотаций (отдельно от блока «Аннотация»). Панель MUST загружать аннотации через `GET /api/nodes/{basePath}/annotations`. На десктопе (lg+) панель MUST отображаться справа от основного контента; на мобильных — через кнопку «Заметки (N)» и bottom sheet. Панель MUST показывать общие и привязанные заметки в одном списке; тело заметки — plain text (`textarea`) с автосохранением.

#### Сценарий: Пустое состояние

- **WHEN** у узла нет аннотаций
- **THEN** панель показывает пустое состояние и кнопку «+ Заметка»

#### Сценарий: Создание общей заметки

- **WHEN** пользователь нажимает «+ Заметка» и вводит текст
- **THEN** UI отправляет `POST /annotations`, заметка появляется в списке

#### Сценарий: Страница перевода

- **WHEN** пользователь открывает перевод узла
- **THEN** отображается та же панель заметок, что и у оригинала

### Requirement: Привязка заметки к выделенному фрагменту

В блоке «Содержание» UI MUST позволять выделить текст и создать привязанную заметку через контекстное действие «Добавить заметку». UI MUST отправлять `POST` с `anchor` (`text_quote`, `content_path` текущего файла, `exact`, `prefix`, `suffix`, опционально `heading_id`). В тексте MUST отображаться маркер (●) у абзацев с успешно разрешёнными (`resolved: true`) привязками в текущем теле.

#### Сценарий: Создание по выделению

- **WHEN** пользователь выделяет фрагмент и сохраняет заметку
- **THEN** создаётся привязанная аннотация, маркер виден у соответствующего абзаца

#### Сценарий: Переход заметка → цитата

- **WHEN** пользователь кликает привязанную заметку с `resolved: true`
- **THEN** страница прокручивается к подсвеченному фрагменту

### Requirement: Устаревшие привязки в UI

Если аннотация имеет `resolved: false`, UI MUST показывать сохранённую цитату из `anchor.exact` и индикатор «привязка устарела». UI MUST предлагать действие «Перепривязать» через повторное выделение текста и `PATCH` якоря.

#### Сценарий: Orphan после изменения тела

- **WHEN** тело узла изменилось и якорь не разрешается
- **THEN** заметка остаётся в списке с пометкой об устаревшей привязке, без подсветки в тексте

### Requirement: Различие аннотации источника и личных заметок

Подписи UI MUST явно разделять блок «Аннотация» (описание источника из frontmatter) и панель «Мои заметки» (личные аннотации пользователя).

#### Сценарий: Страница узла с обоими блоками

- **WHEN** у узла есть frontmatter `annotation` и личные заметки
- **THEN** отображаются оба блока с разными заголовками и назначением
