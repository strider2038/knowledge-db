## Purpose

Личные аннотации пользователя к узлам базы знаний: общие заметки и заметки, привязанные к фрагменту текста. Хранятся отдельно от markdown-тела узла и поля frontmatter `annotation`.

## ADDED Requirements

### Requirement: Sidecar-файл аннотаций узла

Система SHALL хранить личные аннотации узла в файле `{theme}/{slug}/annotations.yaml` (git-tracked). Файл MUST содержать поле `version: 1` и массив `notes`. Каждая запись MUST иметь поля: `id` (UUID v7), `created` (ISO 8601), `updated` (ISO 8601), `body` (plain text). Поле `anchor` MUST быть `null` для общей аннотации или объектом для привязанной аннотации. Файл MUST располагаться у **базового** пути узла (без суффикса языка перевода). Отсутствие файла MUST трактоваться как пустой список аннотаций.

#### Сценарий: Узел без аннотаций

- **WHEN** существует `topic/article.md`, но `topic/article/annotations.yaml` отсутствует
- **THEN** API возвращает пустой список аннотаций без ошибки

#### Сценарий: Общая аннотация

- **WHEN** пользователь создаёт аннотацию без привязки к тексту
- **THEN** в `annotations.yaml` сохраняется запись с `anchor: null`

### Requirement: Формат якоря text_quote

Привязанная аннотация MUST содержать `anchor` с полями: `type: text_quote`, `content_path` (путь файла, в котором создана привязка), `exact` (выделенный текст), `prefix` и `suffix` (контекст до и после, опционально пустые строки), `heading_id` (опционально, slug заголовка секции). Длина `exact` MUST NOT превышать 500 символов; `prefix` и `suffix` — по 80 символов. `body` MUST быть plain text длиной не более 4000 символов.

#### Сценарий: Создание привязанной аннотации

- **WHEN** пользователь выделяет текст в содержании и сохраняет заметку
- **THEN** в anchor сохраняются `content_path`, `exact`, `prefix`, `suffix` и опционально `heading_id`

#### Сценарий: Превышение лимита тела

- **WHEN** `body` длиннее 4000 символов
- **THEN** операция записи возвращает ошибку валидации

### Requirement: Лимиты и идентификаторы аннотаций

Система MUST генерировать `id` как UUID v7 при создании аннотации. На один узел MUST быть не более 200 аннотаций. `created` MUST устанавливаться при создании; `updated` MUST обновляться при каждом изменении `body` или `anchor`.

#### Сценарий: Превышение лимита количества

- **WHEN** у узла уже 200 аннотаций и пользователь создаёт новую
- **THEN** операция возвращает ошибку валидации

### Requirement: Resolve якоря при чтении

При `GET` аннотаций система MUST для каждой привязанной записи вычислять `resolved: boolean`: `true`, если подстрока `exact` найдена в теле файла `content_path` (с учётом `prefix`/`suffix` при неоднозначности); иначе `false`. Заметка MUST сохраняться даже при `resolved: false`.

#### Сценарий: Устаревший якорь после refresh

- **WHEN** тело узла изменилось и цитата `exact` больше не встречается в `content_path`
- **THEN** аннотация возвращается с `resolved: false` и сохранённым `anchor`

#### Сценарий: Успешный resolve

- **WHEN** цитата `exact` найдена в текущем теле `content_path`
- **THEN** аннотация возвращается с `resolved: true`

### Requirement: Общая лента для оригинала и переводов

Запросы аннотаций для пути перевода (`{base}.ru`) MUST возвращать ту же лента, что и для базового пути `{base}`. Запись и изменение MUST выполняться только относительно базового пути.

#### Сценарий: Страница перевода

- **WHEN** пользователь открывает `topic/article.ru` и запрашивает аннотации
- **THEN** API читает `topic/article/annotations.yaml` и возвращает все аннотации узла

### Requirement: Перенос и удаление sidecar

При перемещении узла система MUST переносить `{slug}/annotations.yaml` вместе с директорией вложений. При удалении узла система MUST удалять sidecar. Ingestion и agent-edit MUST NOT изменять `annotations.yaml`.

#### Сценарий: Move с аннотациями

- **WHEN** узел `old/a` с `old/a/annotations.yaml` перемещается в `new/a`
- **THEN** файл оказывается в `new/a/annotations.yaml`

#### Сценарий: Delete узла

- **WHEN** удаляется узел с sidecar аннотаций
- **THEN** `annotations.yaml` удаляется вместе с директорией вложений

### Requirement: Исключение аннотаций из семантического индекса

Содержимое `annotations.yaml` MUST NOT включаться в embedding text, searchable text, hybrid search и RAG-контекст. Изменение только аннотаций MUST NOT изменять `content_hash` узла для embedding-индекса.

#### Сценарий: Добавление заметки без переиндексации

- **WHEN** пользователь добавляет или изменяет только аннотации
- **THEN** `content_hash` узла в embedding-индексе не меняется
