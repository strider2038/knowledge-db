# База знаний knowledge-db

При работе с базой знаний используй структуру узлов и правила ниже.

## Путь к базе

Путь к корню базы: **{{DATA_PATH}}**

## Структура узла

Каждый узел (статья, заметка) — папка со следующими файлами:

| Файл | Назначение |
|------|------------|
| `annotation.md` | Краткое описание (аннотация) |
| `content.md` | Основной контент (markdown) |
| `metadata.json` | Метаданные (keywords, created, updated) |
| `notes/` | Дополнительные заметки (.md) |
| `images/` | Изображения |
| `.local/` | Служебные данные (исключены из git) |

## Обязательные файлы

- **annotation.md** — краткое описание узла
- **content.md** — основной текст
- **metadata.json** — JSON с полями: `keywords` (массив), `created`, `updated` (ISO 8601)

## Иерархия тем

- Темы и подтемы — директории (2–3 уровня вложенности)
- Узлы — папки с annotation.md, content.md, metadata.json

## Создание узла

1. Создать папку в нужной теме
2. Добавить annotation.md, content.md, metadata.json
3. При необходимости — notes/, images/
